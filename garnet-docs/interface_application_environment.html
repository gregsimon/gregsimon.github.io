Detailed Description

An interface for managing a set of applications.

Applications run inside environments, which provide ambient services and support for their lifecycle.

ApplicationEnvironment::CreateNestedEnvironment ( ApplicationEnvironmentHost  host,
ApplicationEnvironment &  environment,
ApplicationEnvironmentController &?  controller,
string?  label 
)

Creates a new environment nested inside this environment.

When applications are created inside the nested environment using the environment's |ApplicationLauncher|, the environment requests the environment services from |host| before passing those services to the newly created application in its |ApplicationStartupInfo|.

Typically the |host| will make the newly created |environment| available in the environment services, giving child applications to launch other applications in the environment and to create nested environments.

The |host| must provide an |ApplicationLoader| service (or inherit one from its parent) if it wishes to allow new applications to be loaded in the nested environment.

The |controller| can be used to control the lifecycle of the created environment. If an |ApplicationEnvironmentController|'s interface is request, the environment will be killed when the interface is closed.

The |label| is an optional name to associate with the environment for diagnostic purposes. The label will be truncated if it is longer than |kLabelMaxLength|.

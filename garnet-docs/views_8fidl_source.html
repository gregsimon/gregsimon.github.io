12 // A view is a graphical user interface component which is responsible
13 // for drawing and supporting user interactions in the area of the screen
14 // that it occupies.
15 //
16 // A view may also act as a container for other views (known as the
17 // view's children) which it may freely layout and position anywhere
18 // within its bounds to form a composite user interface. The hierarchy
19 // of views thus formed is called a view tree.
20 //
21 // LIFECYCLE
22 //
23 // Use |ViewManager.CreateView()| to create a view. The application
24 // uses the |View| interface to manage the view's content and implements
25 // the |ViewListener| interface to handle events.
26 //
27 // To destroy a view, simply close the |View| channel.
28 //
29 // ADDING CHILD VIEWS
30 //
31 // Use |GetContainer()| to obtain an interface for manipulating child views.
32 //
33 // See |ViewContainer| for more information.
34 //
35 // GETTING SERVICES
36 //
37 // The view's |app::ServiceProvider| offers access to many services which
38 // are not directly expressed by the |View| interface itself, such
39 // as input, accessiblity, and editing capabilities.
40 //
41 // For example, perform the following actions to receive input events:
42 //
43 // 1. Call |GetServiceProvider()| to obtain the view's service provider.
44 //
45 // 2. Ask the service provider for its |InputConnection|.
46 //
47 // 3. Set listeners on the input connection to receive events.
48 interface View {
49  // Gets the view's token.
50  GetToken() => (ViewToken token);
51 
52  // Gets a service provider to access services which are associated with
53  // the view such as input, accessibility and editing capabilities.
54  // The view service provider is private to the view and should not be
55  // shared with anyone else.
56  //
57  // See |InputConnection|.
58  GetServiceProvider(app.ServiceProvider& service_provider);
59 
60  // This is a way for an app to offer services that are associated with a
61  // particular view. Used to expose services to the compositor
62  //
63  // In particular this should be used to let know which view is implementing
64  // the ability to show an IME
65  //
66  // |service_names| should contain a list of service names as defined with the
67  // FIDL syntax [ServiceName="service_name"]
68  //
69  // The list of services will be used by the compositor to filter service
70  // providers when looking for a particular service.
71  OfferServiceProvider(app.ServiceProvider service_provider,
72  array<string> service_names);
73 
74  // Gets an interface for managing the view's children.
75  GetContainer(ViewContainer& container);
76 };

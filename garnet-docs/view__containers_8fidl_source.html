10 // A view container is an interface exposed by |View| and |ViewTree| to
11 // manage their child views. Although |View| may have any number of children,
12 // a |ViewTree| can have at most one (its root view).
13 //
14 // EMBEDDING
15 //
16 // The following steps are required to embed another view as a child:
17 //
18 // 1. Obtain the |ViewOwner| belonging to the view you would like to embed.
19 // The means for doing this is not specified by the view system.
20 // You might create another view of your own to embed or connect to
21 // another application using a mechanism such as |ViewProvider| (or
22 // any suitable agreed-upon protocol) to create the view to embed.
23 //
24 // 2. Call |AddChild()| to add the view you would like to embed and assign
25 // it a unique key.
26 //
27 // 3. Call |SetChildProperties()| to provide layout parameters and other
28 // properties for the new child using the same key that was provided
29 // to |AddChild()|.
30 //
31 // 4. Watch for the child becoming unavailable, as reported by
32 // |OnChildUnavailable()|, which indicates that the child is no longer
33 // in a usable state (perhaps the application which provided it has
34 // stopped). When this happens, you are still responsible for calling
35 // |RemoveChild()| to remove the child and discard any state that your
36 // view has associated with it.
37 //
38 // VIEW PROPERTIES AND LAYOUT
39 //
40 // The container controls the presentation of its children by setting
41 // |ViewProperties| for each of them. View properties include layout
42 // information and other parameters the child needs to know to participate
43 // in the view hierarchy.
44 //
45 // The container must set properties for each of its children after adding
46 // them, as otherwise the children cannot be rendered (since they lack enough
47 // context to know what to draw).
48 interface ViewContainer {
49  // Sets the view container listener, or null to remove.
50  SetListener(ViewContainerListener? listener);
51 
52  // Adds the view referenced by |child_view_owner| as a child and assigns
53  // it the provided |child_key| to identify it among its children.
54  // The container may remove the child later by passing the same |child_key|
55  // to |RemoveChild()|.
56  //
57  // This method takes ownership of the view.
58  //
59  // It is important for the container to choose locally unique values for
60  // |child_key| to ensure that each child can be distinguished even as
61  // more children are added or removed. We recommend using a simple
62  // counter which is incremented on each (re-)addition.
63  //
64  // If the child becomes unavailable at any time prior to being removed
65  // then an |OnChildUnavailable()| message will be sent.
66  //
67  // If |child_view_owner| refers to a view which is already unavailable or
68  // if adding the view would create a cycle in the view tree then the
69  // call proceeds as if it succeeded but an |OnChildUnavailable()| message
70  // will be sent.
71  //
72  // If |child_view_owner| refers to a view which already has a container or is
73  // the root of a view tree then an |OnChildUnavailable()| message will
74  // be sent to its old container or root and the the view will be
75  // (re-)added to its new container as usual. This special case also
76  // applies when the specified view is already a child of this view, in which
77  // case the behavior is similar to the view having been transferred to
78  // some other container and then back again.
79  //
80  // Note that an unavailable child will remain in its container's list of
81  // children until its container explicitly calls |RemoveChild()| to remove
82  // it.
83  //
84  // |host_import_token| is an import token which the view manager will
85  // use to import the node to which the child view's content should
86  // be attached.
87  //
88  // To establish the graphical embedding relation, the container view
89  // must create an event pair, bind one endpoint to an |ExportResourceOp|
90  // associated with the node to which the child's content nodes will be
91  // attached as descendants, and pass the other endpoint to this method as
92  // |host_import_token|.
93  //
94  // It is an error to add a view whose |child_key| already appears
95  // in the view's list of children; the connection will be closed.
96  //
97  // It is an error to add more than one child to a |ViewTree|'s container;
98  // it can only have at most one child (its root).
99  AddChild(uint32 child_key, ViewOwner child_view_owner,
100  handle<eventpair> host_import_token);
101 
102  // Removes the view referenced by |child_key| from the view's
103  // list of children.
104  //
105  // If |transferred_view_owner| is not null, associates it with the
106  // previously added child to allow it to be transferred elsewhere or
107  // closes the |transferred_view_owner| channel if there was none.
108  //
109  // It is an error to remove a view whose |child_key| does not appear
110  // in the container's list of children; the connection will be closed.
111  RemoveChild(uint32 child_key, ViewOwner&? transferred_view_owner);
112 
113  // Sets view properties for the child, such as layout constraints.
114  //
115  // This method must be called at least once after a child is added to
116  // set the view's properties before it can be rendered. Rendering for
117  // children without properties is blocked until properties are set.
118  //
119  // The child will receive the new properties along with a scene version
120  // which it should apply to the first scene update which includes the
121  // effects of the property change.
122  //
123  // The |child_scene_version| specifies the version number which
124  // the child should use when publishing scene updates which incorporate
125  // the new properties. May be |kSceneVersionNone| if the version number
126  // is not important (meaning that there is no need to coordinate the
127  // scene updates with the child).
128  //
129  // The |child_view_properties| specifies the properties for the child, or
130  // null to remove the properties from the child which will cause rendering
131  // of the child's scene to be blocked until new properties are set.
132  //
133  // It is an error to specify a |child_key| that does not appear in
134  // the container's list of children; the connection will be closed.
135  //
136  // It is an error to specify malformed |child_view_properties| such
137  // as invalid layout properties; the connection will be closed.
138  SetChildProperties(uint32 child_key, ViewProperties? child_view_properties);
139 
140  // Request the child view to get input focus for key events.
141  RequestFocus(uint32 child_key);
142 };

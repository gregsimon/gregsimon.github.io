12 // The input connection service allows a view to receive input events
13 // such as key presses and pointer movements.
14 
15 // This service can be retrieved from the |View| service provider.

17 // TODO(jeffbrown): Elaborate this. Particularly need to think about

22  //
23  // If |listener| is null, then the previously set listener is removed.

34 // An interface applications may implement to receive events from an
35 // input connection.
36 // TODO(MZ-33): This should include hit testing and an input arena.
37 interface InputListener {
38  // Called to deliver an input event.
39  //
40  // The |consumed| result should indicate whether the input event was
41  // consumed by the connection. If |consumed| is false, the system may
42  // apply default behavior of its own in response to the event.

44 };
